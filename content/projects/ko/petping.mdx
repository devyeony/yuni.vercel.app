## 📌 프로젝트명
- **펫핑 (Petping)**

## 📋 프로젝트 개요
- <b>펫핑 (Petping)</b>은 **반려동물**의 **산책**, **영양**, **쇼핑**, **보험**을 **통합 관리**할 수 있는 **올인원 헬스케어 앱**입니다.
- 사용자와 반려동물의 **건강한 라이프스타일**을 지원하기 위한 **다양한 기능**을 제공하며, 앱 전반의 **사용자 경험**과 **서비스 운영 효율성**을 지속적으로 개선해 왔습니다.

## 📅 기간
- 2021년 7월 – 2024년 1월

## 🛠 기술 스택
- **백엔드:** Java, Spring, Spring Boot, JPA, MyBatis
- **프론트엔드:** JavaScript, HTML, CSS
- **데이터베이스:** MySQL, PostgreSQL, Redis
- **DevOps:** Docker, Jenkins, Linux
- **클라우드 인프라 (AWS 서비스):** EC2, S3, ECS, ElastiCache, CodeDeploy, IAM, Route 53
- **모니터링 & 로깅:** Grafana, Loki
- **데이터 시각화 & BI 도구:** Redash

## 👤 역할 및 기여
- **백엔드 전반 아키텍처 및 운영 주도:** 백엔드 아키텍처 설계 및 서비스 개발·배포·운영까지 전 과정 담당. 인프라 관리와 개발 프로세스 개선에도 능동적으로 기여
- **크로스펑셔널 협업 중심의 팀 플레이어:** PM, 디자이너, 프론트/앱 개발자, QA 등 다양한 직군과 협업하며 원활한 커뮤니케이션과 기획 이해를 바탕으로 기능 개발에 적극 참여
- **데이터 기반 개발에 대한 관심과 실천:** 마케터, 데이터 엔지니어와 긴밀히 소통하며 데이터 구조 설계와 수집 흐름에 주도적으로 기여, 데이터 기반 서비스 개선에 지속적으로 관심을 갖고 헌신
- **고객 중심 문제 해결에 대한 확고한 책임감:** 서버 로그와 데이터를 바탕으로 고객 문제와 요구사항을 파악하고, 해결책과 응대 방향을 제시해 CS팀이 효과적으로 대응할 수 있도록 지원. 필요 시 해당 기술 담당자와 원활히 연결하여 문제 해결까지 책임지고 조율
- **역할을 넘어 팀과 서비스 전체 관점에서 주도적 참여:** 자신의 역할에 한정하지 않고, 서비스 전반에 관심을 갖고 아이디어 제안, 기획 피드백, 테스트와 배포 후 모니터링까지 꾸준히 수행. 고객 피드백과 데이터를 꾸준히 살피며 문제를 조기에 발견하고 팀과 협력해 개선하는 데 앞장섬

## ✨ 주요 기능
| **기능** | **설명** |
|-------------|---------|
| **펫 케어 대시보드** | 펫 건강 관련 데이터 요약, 날씨 및 미세먼지 정보 제공 |

| **영양 기능** | 사료 검색, 식사 기록, 칼로리 분석, 반복식단 관리, 알러지 관리 |
| **건강 캘린더** | 산책·식단 통합 캘린더, 건강 다이어리 관리, 건강 수치 기록, 건강 기록 소셜 공유 |
| **미션 및 리워드** | 출석·산책·이벤트 미션, 포인트 적립·사용·만료 |
| **푸시 및 알림 센터** | 유저 활동 기반 자동 푸시, 미션 진행·완료 알림, 앱 내 인앱 알림 제공 |
| **커머스 기능** | 쇼핑몰 연동 및 포인트 사용, 추천 상품 및 유저 맞춤형 상품 배너 노출 |

## 🚀 기술적 도전 및 해결

### 🧪 QA 엔지니어 편의를 위한 테스트용 API 및 데이터 조회 쿼리 제공  
**문제:** QA 테스트 과정에서 더미 데이터 생성이 반복적으로 필요했으며, 데이터 조회가 어려워 다양한 시나리오 테스트가 어려웠음

**해결**  
- QA 전용 테스트용 API 구현으로 더미 데이터 자동 생성 지원  
- Redash에 QA용 데이터 조회 및 검증 쿼리문 작성·공유  
- QA 엔지니어들이 스스로 다양한 테스트 시나리오를 손쉽게 수행할 수 있도록 환경 제공

---

### 📊 데이터 기반 의사결정을 위한 실시간 데이터 조회 지원  
**문제:** 데이터 엔지니어 및 마케터가 비즈니스 지표와 데이터를 신속하게 확인하기 어려움

**해결**  
- Redash에 실시간 데이터 조회 쿼리 작성 및 공유  
- 다양한 비즈니스 인사이트와 지표 제공으로 데이터 기반 의사결정 지원  
- 데이터 협업 문화 조성 및 부서 간 커뮤니케이션 활성화

---

### 📈 Jira 자동화와 연동으로 개발 프로세스 및 배포 현황 실시간 관리  
**문제:** 개발 이슈 관리와 배포 현황 공유에 수작업이 많고 커뮤니케이션 지연 발생

**해결**  
- Jira Automation 설정으로 백엔드 이슈를 칸반보드로 자동 관리  
- Confluence에 자동으로 릴리즈 노트 생성 및 Slack 채널로 실시간 공유  
- 개발 프로세스 효율성 향상과 전사 배포 현황 투명성 확보

---

### 🚀 CI/CD 파이프라인 구축 및 블루/그린 배포 도입  
**문제:** 수동 배포로 인한 배포 오류 및 서비스 다운타임 발생 위험

**해결**  
- Jenkins와 Bitbucket 연동으로 자동 빌드·배포 파이프라인 구성  
- Docker Compose, ECS, CodeDeploy 기반 자동 배포 환경 구축  
- 펫보험 운영 서버에 블루/그린 배포 적용으로 무중단 배포 실현

---

### 📚 JPA 도입 및 혼용 환경 구축으로 데이터 접근 효율성 제고  
**문제:** 기존 MyBatis 단독 사용으로 인한 개발 생산성 및 유지보수 어려움

**해결**  
- 기존 시스템에 JPA 도입, MyBatis와의 혼용 환경을 유연하게 구성
- 팀 내 JPA 스터디에 적극 참여하며 기술 습득 → 실무에 빠르게 적용
- 도메인별 쿼리 최적화 및 코드 간결화를 통해 유지보수성과 데이터 접근 효율 향상

---

### 🤝 외주 협업 주도 및 커머스 서비스 연동 관리  
**문제:** 펫 용품 쇼핑몰 개발을 외주에 맡기면서 연동 및 API 관리 어려움

**해결**  
- 쇼핑몰과 서비스 리워드 연동 API 설계·구현 및 공유 전담  
- 외주업체와 개발 관련 피드백, 요구사항 관리 및 일정 조율 주도  
- 원활한 협업과 품질 높은 서비스 연동 성공적 완수

---

### 🐾 펫보험 청구 프로세스 자동화로 운영 리소스 절감 및 회원 만족도 제고  
**문제:** CS팀의 수동 청구 접수와 보험사 전달로 처리 지연과 업무 과중 발생

**해결**  
- 청구 정보 저장, 청구서 PDF 자동 생성 및 첨부파일 관리 백엔드 구현  
- 보험사 청구 담당자에게 이메일 자동 전송 및 회원, CS팀, 보험사에 메일·문자 알림 발송  
- 업무 효율성 대폭 향상과 회원 처리 지연 최소화

---

### 📜 로그 중앙화 및 실시간 모니터링 시스템 구축  
**문제:** 로그 확인이 수동이고 서버 인스턴스별 분산으로 로그 관리 어려움

**해결**  
- API 요청·응답 로그를 별도 DB 테이블에 저장  
- Grafana, Loki를 활용한 로그 중앙화 및 실시간 모니터링 환경 구축  
- 다중 서버 환경에서 신속한 장애 대응과 서비스 안정성 강화

---

### ⚠️ 외부 API 장애 자동 감지 및 장애 알림 체계 확립  
**문제:** 공공데이터포털 동물등록번호 API 서버 장애 시 서비스 장애 조기 인지 어려움

**해결**  
- API 호출 실패 시 다회 재시도 로직 구현  
- 재시도 후에도 실패 시 Slack 자동 장애 알림 발송  
- 장애 메시지 커스터마이징과 장애 리포트 프로세스 구축으로 빠른 대응 체계 마련

---

### 🎉 대규모 사용자 급증 상황에서 안정적 서비스 운영 및 신속 대응  
**문제:** 외부 행사 공식 앱 사용 중 급격한 사용자 증가로 서버 부하 및 장애 우려

**해결**  
- 강아지 산책 챌린지 이벤트 백엔드 안정적으로 구현 및 운영  
- 실시간 모니터링과 1:1 문의 대응으로 장애 최소화 및 사용자 만족도 유지  
- 이벤트 기간 내 지속적 서버 상태 관리 및 문제 발생 시 즉각 대응 체계 운영

## 📈 성과
- 📈 **MAU 30% 증가:** 산책 이벤트 및 외부 행사 기능을 안정적으로 개발·운영해 사용자 참여 및 월간 활성 사용자 30% 증가 견인  
- 🔔 **푸시 자동화로 DAU 20% 상승:** 행동 기반 리마인드·리워드 알림 도입으로 일일 활성 사용자 증가  
- 📉 **보험 문의 20% 감소:** 청구서 자동 생성 및 보험사 연동 메일 발송 프로세스 구현으로 CS팀 업무량 감소
- ⚙️ **운영 처리 속도 50% 향상:** Redash 쿼리 자동화로 통계 업무 효율화  
- 📊 **테스트 커버리지 40% 증가:** QA 전용 API 및 더미 데이터 자동 생성 도입  
- 🚀 **무중단 배포율 95% 이상 달성:** 블루/그린 배포로 안정적 무중단 배포 실현  
- 🔎 **장애 탐지 시간 60% 단축:** 로그 중앙화와 실시간 모니터링, 외부 API 자동 장애 알림 체계 구축으로 신속 대응 강화
- 💬 **Jira·Slack 자동화로 커뮤니케이션 지연 70% 감소:** 이슈·배포 알림 자동화로 공유 속도 향상  
- 🤝 **외주 협업 안정화 및 커머스 연동 90% 완성도 달성:** 외부 개발사 소통과 일정 조율 주도, 원활한 협업 관리
