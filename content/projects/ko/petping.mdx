펫핑은 산책, 식사, 건강, 보험, 쇼핑을 한곳에서 관리하는 반려동물
라이프스타일 앱이다. 백엔드 엔지니어로 합류해 2년 반 동안 서버 사이드의
대부분 — API 설계, 그 위에서 돌아가는 AWS 인프라, 서비스를 건강하게 유지하는
운영 도구 — 를 책임지게 되었다.

## 만들기

도메인은 빠르게 자랐다 — 실시간 이벤트가 있는 산책 트래킹, 식사 기록, 외부
보험사와 연동하는 보험 청구, 커머스 연동까지. 영속성 계층이 첫 구조적
결정이었다. 코드베이스는 MyBatis 중심이었고, 전면 재작성은 기능 개발을 몇 달간
멈춰야 했다. 신규 도메인에 JPA를 도입하고 검증된 쿼리 중심 경로는 MyBatis로
유지하는 하이브리드 구조는, 마이그레이션에 제품을 걸지 않으면서 엔티티 수준의
생산성을 얻는 선택이었다.

보험 청구는 가장 깊은 연동 작업이었다. 수동이던 청구 흐름을 외부 보험사 API에
맞춰 자동화하고 그 위에 커머스 API 연동을 얹었다. 성공의 척도는 지연 시간
그래프가 아니라 CS 큐였다 — 보험 관련 문의가 5분의 1 줄었다.

## 운영하기

매일 쓰는 반려동물 앱은 배포 다운타임을 감당할 수 없다. CI/CD 파이프라인을
AWS(ECS, CodeDeploy) 기반 블루/그린 무중단 배포까지 구축했다. 가시성 측면에서는
자체 호스팅한 Grafana + Loki 스택으로 구조화 로그를 대시보드와 알림으로
흘려보내고, 실제 장애 패턴에 맞게 튜닝해 탐지-대응 시간을 절반 이하로 줄였다.
Redash 뷰로 운영팀이 통계를 셀프서브로 조회하게 하고 QA에는 전용 테스트 API를
제공했다 — 둘 다 백엔드 팀을 다른 팀 워크플로의 병목에서 제거한 작업이다.
