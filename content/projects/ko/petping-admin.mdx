## 📌 프로젝트명
- **펫핑 어드민 (Petping Admin)**

## 📋 프로젝트 개요
- <a href="/ko/projects/petping" target="_blank"><b>펫핑</b></a> 서비스의 **내부 운영** 및 **고객 응대**를 위한 **백오피스 시스템**을 **풀스택**으로 개발했습니다.
- <b>‘펫핑 어드민 (Petping Admin)’</b>은 콘텐츠, 사용자, 리워드, 이벤트 등을 효율적으로 관리할 수 있는 **내부 운영 도구**로, **운영팀의 업무 효율 향상**을 목표로 설계되었습니다.

## 📅 기간
- 2021년 7월 – 2024년 1월

## 🛠 기술 스택
- **백엔드:** Java, Spring Boot, Spring Security, JPA, MyBatis
- **프론트엔드:** JavaScript, HTML, CSS
- **템플릿 엔진:** Thymeleaf
- **데이터베이스:** PostgreSQL
- **UI 프레임워크 및 라이브러리:** Bootstrap, DataTables.js, Chart.js

## 👤 역할 및 기여
- 기능 효율성과 사용성을 중심으로 **기획·개발·유지보수 전 과정을 수행**, **풀스택 개발** 주도
- **전체 시스템 아키텍처**를 **직접 설계 및 구현:** 백엔드, 프론트엔드, UI/UX, 데이터베이스 포함
- 운영팀의 실제 업무 흐름과 문제점을 분석해 **비즈니스 요구사항**을 **주도적으로 정의**하고 **기능화**
- 펫핑 서비스 운영 중 반복되는 **수동 업무를 자동화**하여 **실질적인 운영 효율성 향상**

## ✨ 주요 기능
| **기능**                     | **설명** |
|----------------------------------|----------|
| **사용자 및 권한 관리**           | 어드민 권한 담당자 전용 기능으로, 세분화된 권한 설정과 접속 로그 조회를 통해 안정적인 운영 환경 제공 |
| **서비스 접속 로그 관리**         | 펫핑 회원의 서비스 이용 이력을 기반으로 버그 추적, 기능별 사용 현황 및 사용자 반응 분석에 활용 |
| **회원 및 반려동물 정보 관리**     | 펫핑 회원 및 반려견 프로필 데이터를 CRUD 기반으로 체계적 관리 |
| **콘텐츠 및 게시판 관리**         | 공지사항, FAQ, 게시글, 팝업/배너 등 다양한 콘텐츠를 효율적으로 운영 |
| **산책 및 위치 데이터 관리**       | 산책 기록 조회, 오디오 가이드 및 장소 POI 데이터 관리 기능 포함 |
| **미션 및 리워드 관리**           | 이벤트 및 미션 운영과 회원별 미션 현황 모니터링을 통해 회원 참여 증진 |
| **보험 청구 관리**               | 계약자 및 청구 내역 관리와 청구 관련 이메일 설정 기능을 통해 펫보험 처리 업무 지원 |
| **쇼핑몰 운영 관리**             | 펫핑샵 회원 목록, 리워드 내역, 이벤트 배너, 추천 상품 관리 등 쇼핑몰 핵심 기능 통합 관리 |
| **메세지 발송 및 고객 소통**       | 이메일, 푸시, 카카오 알림톡 발송과 1:1 문의 응대를 통해 원활한 고객 커뮤니케이션 지원 |
| **운영자 UI/UX 최적화**          | 관리자 대시보드 구성, 검색·필터링·일괄 처리 기능을 통해 운영 편의성 강화 |

## 🚀 기술적 도전 및 해결

### 🔐 보안 강화된 계정 및 권한 관리 체계 전환
**문제:** 공유된 관리자 계정으로 인한 심각한 보안 및 감사 문제

**해결**  
- **Spring Security** 기반 맞춤 인증·인가 시스템 구축 
- **Google OAuth2** 연동으로 **사내 계정 로그인 제한**, **Google Authenticator 기반 OTP 2단계 인증** 도입  
- 역할별 접근 제어 및 메뉴 조건부 노출 구현

---

### ⚙️ 단독 개발에 최적화된 유지보수 아키텍처 선택
**문제:** 프론트엔드/백엔드 분리 구조가 유지보수 부담이 크고, **협업 리소스가 부족**한 상황에서 **전체 프로젝트를 혼자** 운영해야 했음

**해결**  
- **Thymeleaf SSR 아키텍처** 채택으로 Spring Boot와 긴밀한 통합  
- 프론트엔드 빌드 과정 생략, 배포 복잡성 및 유지보수 부담 완화

---

### 📞 고객 지원 품질 향상을 위한 API 로그 뷰어 도입  
**문제:** 외부 로그 툴(Airbridge)로는 사용자별 API 흐름 추적 한계

**해결**  
- 어드민 내 사용자별 API 호출 내역 조회 기능 구현  
- 신속한 디버깅과 정확한 고객지원 가능

---

### 🗺️ 산책 데이터 시각화로 이벤트 운영 효율 강화  
**문제:** **위치 정보 시각화 부재**로 이벤트 검증 및 버그 추적 비효율

**해결**  
- **네이버 지도 API** 연동, **산책 경로 및 POI 마킹 정보 시각화** 구현  
- 운영팀의 직관적 확인과 신속 대응 가능

---

### 🤖 반복 수작업 자동화를 통한 운영 효율 개선
**문제:** 일부 예외 상황에서 미션 달성 처리와 리워드 지급을 **SQL 쿼리로 수동 실행**해야 하는 경우가 발생해 업무 지연

**해결**  
- 운영자가 직접 사용할 수 있는 **UI 기반 수동 처리 기능**을 어드민에 구현  
- 운영팀이 **개발팀 개입 없이도 예외 처리 업무를 실시간으로 수행**할 수 있어 **대응 속도와 정확성**이 크게 향상됨

---

### 📧  보험 청구 이메일 발송 프로세스의 유연성 확보
**문제:** **이메일 누락/오류**가 발생해도 **운영자가 직접 확인·조치할 방법 부족**

**해결**  
- 어드민에서 **청구 PDF·첨부파일 열람/다운로드** 기능 구현  
- **이메일 재발송**, **수신자 주소 수정 기능** 제공 → 운영자의 독립적 대응 지원, 개발자 의존도 감소 
- **보험사와의 커뮤니케이션 정확도** 및 **시스템 신뢰도** 향상

---

## 📈 성과
- 🚀 **운영 효율 200% 향상:** 반복 업무 자동화 및 통합 UI 도입으로 운영팀의 일일 업무 처리량이 2배 이상 증가
- 😊 **운영자 만족도 30% 향상:** 직관적인 UI, 강력한 검색·필터링 기능, 상세한 에러 메시지 도입으로 내부 운영팀의 업무 만족도가 약 30% 증가
- ⚡ **고객 응대 시간 40% 단축:** 통합 사용자 정보와 문의 이력 조회 기능으로 CS 처리 속도가 평균 40% 빨라짐
- 🎯 **리워드 소진율 85% 유지:** 실시간 모니터링 기능을 통해 적시 리워드 지급과 마케팅 대응으로 85% 이상의 리워드 소진율 달성
- 🗺️ **이벤트 운영 시간 25% 단축:** 위치 기반 산책 데이터 시각화로 운영팀의 데이터 확인 및 대응 시간이 25% 감소
- 🔐 **관리자 계정 보안 강화:** Google OAuth2 및 OTP 2단계 인증 도입 후 보안 강화로 무단 접근 위험이 현저히 낮아짐
